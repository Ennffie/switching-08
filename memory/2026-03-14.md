## 今日遇到的關鍵問題

### 1. GitHub Pages 再次出現白畫面錯覺
**現象：**
- 使用者多次回報 `https://ennffie.github.io/switching-07/` 變成空白畫面。

**實際情況：**
- 經 curl + browser snapshot 驗證，主頁多次都正常 render。
- 問題高機率仍然係：
  - Telegram / iPhone 內置 browser cache
  - in-app webview 狀態異常
  - 外部網站在 webview 內顯示不穩定

**教訓：**
- 每次收到「空白畫面」回報，唔好即刻假設 build 壞咗。
- 先做三步驗證：
  1. `curl` 抓最新 HTML
  2. browser console 檢查 runtime error
  3. snapshot 檢查有冇主頁內容
- 如果三樣正常，先考慮係 cache / webview 問題。

---

### 2. 底部第三格 badge 一度唔統一
**現象：**
- 有啲頁第三格有數字 badge，有啲頁冇。

**根因：**
- repo 入面一度同時存在兩套底部導航：
  - 舊 hardcoded nav（OverviewPage / MyMPFPage）
  - 新 shared `BottomNav`（Todo / Profile）

**解法：**
- 最後統一改用 shared `BottomNav`
- badge count 由 shared component 控制

**教訓：**
- 類似底部導航呢種 cross-page UI，一定要抽 component。
- 如果唔抽，後面必然會出現「A 頁改咗、B 頁未改」問題。

---

### 3. Step 1 外部連結 click 範圍錯誤
**現象：**
- 用戶發現成張計劃卡都會觸發 external link，唔只係右邊 link icon。

**根因：**
- 內層連結區塊 click 區域太大，影響整張卡行為。

**解法：**
- 現在分離成兩種 click：
  - 整張卡：只做計劃選擇
  - 只有 external link icon：才打開外部連結

**教訓：**
- 卡片本身有 selection 行為時，內部 secondary action 必須嚴格限制 hit area。
- 尤其係 mobile UI，唔好令大區域誤觸外部行為。

---

### 4. AIA 外部網站在 iPhone / Telegram webview 有白畫面問題
**現象：**
- 用戶撳 AIA 外部連結時，喺 iPhone/Telegram 內出現白畫面。

**檢查結果：**
- 實際 AIA 連結並非失效；在桌面 browser 可正常開啟。
- 頁面內甚至有 cookie / important notice modal。
- 問題係外站在 in-app browser / webview 顯示不穩定。

**教訓：**
- 如果 production flow 要穩定，應優先考慮：
  - 外站內容本地化到 app 內頁
  - 或最少將外站當 optional 而非主流程依賴
- 之後如再遇到類似情況，可考慮把外部基金列表改成本地頁。

---

### 5. Step 2 個人資料頁要跟 reference，而唔係「功能先」
**現象：**
- 用戶多次強調要「跟足」「照 copy」，包括：
  - 下半部展開卡
  - CRS 區塊
  - 已填值狀態
  - 細 iAM Smart icon 要換成指定版本

**教訓：**
- 呢個 project 係高保真 mock / prototype，不是純 functional demo。
- 後續處理原則：
  1. 先跟版面
  2. 再補互動
  3. 最後先考慮抽象化/重構
- 唔可以再用「大概似」心態做；要當成 pixel-sensitive UI work。
